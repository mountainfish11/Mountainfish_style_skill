## 指南（≤20 条，会话启动注入摘要）

### 嵌入式固件中避免使用 printf/stdio  `<!-- tier: guideline -->`

**原则**: 嵌入式固件（尤其是 FreeRTOS/裸机）中不要用 `printf()` 做日志输出，应使用项目专用的日志宏。

**做法**: 用 UART/DMA 驱动的专用日志宏（如 `LOG_DBG()`、`LOG_INFO()`），而非 `printf()`+`stdio.h`。printf 依赖系统调用、阻塞时间长、在多任务环境下不是线程安全的。如需格式化字符串，用 `snprintf()` + 专用输出函数。

**适用**: `#C #嵌入式 #FreeRTOS #调试 #日志`
